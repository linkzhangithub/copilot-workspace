# 声明项目的技术栈和核心API规范
---

## 1. Vue 3 前端规范
- 使用组合式 API（`<script setup>`）。
- Props 必须定义类型和默认值。
- `v-for` 必须提供唯一的 `key`。
- 严禁同一元素上同时使用 `v-if` 和 `v-for`。
- `<style>` 必须使用 `scoped`。
- 状态管理：Pinia 或 `ref`/`reactive`。

## 2. Node.js 后端规范
- 所有 `async/await` 必须被 `try-catch` 包裹。
- 所有外部输入必须校验。
- **API 统一响应格式**：成功 `{ success: true, data }`，失败 `{ success: false, error }`。
- 禁止硬编码敏感信息，使用环境变量。
- 禁止在主线程中使用同步阻塞方法（如 `fs.readFileSync`）。

## 3. 并行开发与协作协议
在前后端分离项目中，执行以下并行开发协议：
- **契约优先**：后端必须先定义 API 接口 (Swagger/YAPI/Markdown)，前端基于 Mock 数据开发。
- **互不阻塞**：后端实现业务逻辑时，不得等待前端页面完成；前端开发 UI 时，不得等待后端接口完成。
- **集成指令**：当双方都完成后，使用 `@checker` 指令进行集成测试。

## 4. 编码与输出规范
- **语言**：代码注释和文档使用中文。
- **代码风格**：
    - 变量命名语义化 (英文)。
    - 函数必须有 JSDoc/DocString 注释，说明参数、返回值和异常。
    - 严禁硬编码敏感信息 (API Keys)，必须使用环境变量。
- **终端执行**：
    - 涉及文件操作或服务器启动，优先使用项目配置的 Shell (如 Git Bash)。
    - 遇到报错，先分析日志，自动尝试修复，不要直接报错退出。
